# POMOKU



POMOKU는 트럼프 카드와 오목 규칙을 결합한 2D 전략 카드 보드게임입니다.



플레이어는 손패에 있는 카드를 사용해 10×10 보드 위의 동일한 카드 칸에 칩을 배치합니다.

같은 팀의 칩 5개를 가로, 세로, 대각선 방향으로 연결하면 포목이 완성되며, 완성된 카드 조합에 따라 점수를 획득합니다.



이 프로젝트의 최종 목표는 단순한 프로토타입 제작이 아니라, PC용 완성 게임을 제작하고 Steam에 출시하는 것입니다.



---



## 최종 목표



POMOKU의 최종 목표는 다음과 같습니다.



1. 로컬 환경에서 핵심 게임 룰 완성

2. 플레이 가능한 2D UI 구현

3. 4인 / 6인 팀 기반 멀티플레이 구현

4. 특수 카드와 족보 점수 시스템 완성

5. 온라인 방 생성, 방 참가, 친구 초대 기능 구현

6. 게임 연출, 사운드, 튜토리얼, 옵션 추가

7. Steamworks 연동

8. Steam 스토어 페이지 준비

9. 테스트 빌드 검수

10. PC / Steam 정식 출시



---



## 개발 환경



* Engine: Unity 6.4

* Template: Universal 2D

* Language: C#

* Target Platform: PC / Steam

* Version Control: Git



---



## 개발 방향



이 프로젝트는 최종적으로 Steam 출시를 목표로 하지만, 처음부터 모든 기능을 동시에 구현하지 않습니다.



POMOKU의 핵심은 그래픽보다 룰, 턴, 카드, 보드, 점수, 멀티플레이 동기화에 있습니다.

따라서 먼저 로컬 환경에서 게임 규칙을 안정적으로 구현한 뒤, 온라인 멀티플레이와 Steam 기능을 단계적으로 추가합니다.



---



## 개발 단계



### Phase 1. 로컬 MVP



목표는 멀티플레이 없이 한 컴퓨터 안에서 게임의 핵심 룰이 정상적으로 작동하는 것입니다.



구현할 기능:



* 10×10 보드 생성

* 카드 데이터 구조 작성

* 덱 생성 및 셔플

* 플레이어 손패 분배

* 손패 카드 선택

* 선택한 카드와 일치하는 보드 칸 강조

* 보드 칸 클릭 시 칩 배치

* 포목 완성 검사

* 기본 점수 계산

* 턴 진행

* 최소한의 보드 / 손패 / 점수 UI



---



### Phase 2. 룰 확장



로컬 MVP가 안정화되면 POMOKU의 핵심 전략성을 추가합니다.



구현할 기능:



* 스트레이트 포목

* 플러시 포목

* 트리플 포목

* 투페어 포목

* 무적 Jari 포목

* J 카드 특수 효과

* 조커 카드 효과

* 포목 잠금 처리

* 게임 종료 조건

* 동점 처리 규칙



---



### Phase 3. UI / UX 개선



게임을 실제 플레이 가능한 형태로 다듬습니다.



구현할 기능:



* 메인 메뉴

* 로비 화면

* 게임 HUD

* 턴 표시

* 팀 점수 표시

* 남은 덱 수 표시

* 배치 가능한 칸 강조

* 카드 사용 연출

* 포목 완성 연출

* 결과 화면

* 옵션 메뉴



---



### Phase 4. 온라인 멀티플레이



POMOKU의 최종 형태인 팀 기반 멀티플레이 구조를 구현합니다.



구현할 기능:



* 2 vs 2 모드

* 2 vs 2 vs 2 모드

* 방 생성

* 방 참가

* 플레이어 준비 상태

* 팀 배정

* 서버 권한 기반 룰 처리

* 보드 상태 동기화

* 턴 동기화

* 점수 동기화

* 손패 비공개 처리



---



### Phase 5. Steam 출시 준비



게임이 안정적으로 플레이 가능해지면 Steam 출시를 준비합니다.



구현 및 준비할 기능:



* Steamworks 계정 준비

* Steam 앱 등록

* Steamworks SDK 연동

* Steam 친구 초대

* Steam 로비 연동

* Steam 빌드 업로드

* 스토어 페이지 작성

* 게임 설명문 작성

* 스크린샷 제작

* 트레일러 제작

* 최소 사양 / 권장 사양 정리

* 테스트 빌드 검수

* 출시 체크리스트 작성

## 현재 가장 먼저 구현할 기능



현재 우선순위는 로컬 MVP입니다.



가장 먼저 구현할 기능은 다음과 같습니다.



```text

10×10 보드를 생성하고,

각 칸에 카드 정보를 배치한 뒤,

화면에 보드 셀을 표시하는 기능

```



그 다음 순서는 다음과 같습니다.



1. 카드 데이터 구조 작성

2. 보드 셀 데이터 구조 작성

3. 10×10 보드 생성

4. 보드 UI 표시

5. 덱 생성 및 셔플

6. 손패 지급

7. 카드 선택

8. 칩 배치

9. 포목 검사

10. 점수 계산

11. 턴 진행

## Codex 작업 지침



Codex는 이 프로젝트를 단순 프로토타입이 아니라 Steam 출시를 목표로 하는 게임 프로젝트로 다룹니다.



작업 시 다음 원칙을 따릅니다.



1. README.md와 Docs 폴더의 기획서를 먼저 읽습니다.

2. 현재 단계가 어느 Phase인지 확인합니다.

3. 한 번에 모든 기능을 구현하지 않습니다.

4. 먼저 로컬 MVP를 안정적으로 완성합니다.

5. 이후 룰 확장, UI 개선, 멀티플레이, Steam 연동 순서로 진행합니다.

6. 초보자도 이해할 수 있도록 코드 구조를 단순하고 명확하게 유지합니다.

7. 파일을 수정한 뒤 어떤 파일을 변경했는지 요약합니다.

8. 임시 테스트 코드는 최종 구조에 섞이지 않도록 분리합니다.

9. Steam 출시를 고려하여 입력, 해상도, 빌드 설정, 저장 구조를 점진적으로 정리합니다.

POMOKU는 손패의 카드를 사용해 보드 위의 카드 칸에 칩을 놓고, 팀 단위로 5개의 칩을 연결해 점수를 얻는 2D 전략 카드 보드게임이며, 최종 목표는 PC / Steam 정식 출시입니다.
